# bareun.vercel.app 원본 구현용 명세 (공개 관찰 기반)

> 대상: https://bareun.vercel.app/  
> 관찰 원칙: 공개적으로 확인 가능한 화면/문구/상호작용만 기반으로 작성 (원본 코드 재사용·복사 없음)

---

## 1) 페이지 수 / 주요 화면

단일 페이지 앱(SPA) 구조로 보이며, 내부 상태 전환으로 여러 화면이 바뀌는 형태다.

주요 화면(상태) 후보:
1. 메인/홈(대여·반납·관리 진입)
2. 대여 - 물품 선택 (`borrow_select`)
3. 대여 - 수량 조정 (`borrow_qty`)
4. 대여 - 사용자 정보 입력 (`borrow_user`)
5. 반납 - 반납 대상 선택 (`return_select`)
6. 관리자 홈 (`admin_home`)
7. 관리자 - 물품 마스터 관리 (`admin_equipments`)
8. 관리자 - 전체 대여 기록 (`admin_history`)
9. 관리자 - 비밀번호 설정 (`admin_settings`)
10. 성공 오버레이 (`success`)

---

## 2) 섹션 순서

### A. 대여 플로우
1. **물품 선택 섹션**
   - 카드형 물품 그리드(이미지/이름/남은 수량)
   - 선택된 카드 체크 표시
   - 대여 불가 상태(비활성) 표현
2. **다음 단계 고정 하단 CTA**
   - 선택이 있어야 활성화
3. **수량 조정 섹션**
   - 선택 물품별 스테퍼(감소/증가)
   - 최대 가능 수량 안내
   - 수량 추적 비활성 물품은 1개 고정 안내
4. **사용자 입력 섹션**
   - 탭: `학년/반 선택` vs `직접 입력`
   - (선택 모드) 학년/반 드롭다운
   - (직접 입력 모드) 텍스트 입력
   - 대여 요약 박스(선택 물품과 수량)
5. **대여 완료 버튼**
6. **성공 화면 오버레이 후 메인 복귀**

### B. 반납 플로우
1. 현재 대여 중 목록 표시
2. 항목 다중 선택
3. 하단 고정 CTA로 선택건 반납 처리
4. 대여 중 항목이 없으면 "모두 반납됨" 빈 상태 표시

### C. 관리자 플로우
1. 관리자 홈 카드 3개
   - 물품 마스터 관리
   - 전체 대여 기록
   - 관리자 설정
2. 물품 관리
   - 새 물품 추가(사진/이름/수량/수량입력옵션)
   - 등록 물품 리스트 + 삭제
3. 전체 기록
   - 대여중/반납완료 상태 배지
   - 시간/물품/수량/대여자 표시
4. 관리자 설정
   - 새 비밀번호 + 확인 입력

## 3) 주요 버튼과 입력 요소

### 공통 버튼
- `다음 단계로`
- `대여 완료하기`
- `선택한 N건 반납 완료`
- `추가하기`(물품 추가)
- `변경하기`(관리자 비밀번호)

### 대여 관련 입력
- 물품 카드 선택 토글 버튼
- 수량 증감 버튼(- / +)
- 학년 select (1~6)
- 반 select (1~6)
- 직접 입력 text field (예시 placeholder: 이름/부서)

### 반납 관련 입력
- 반납 대상 항목 선택 토글

### 관리자 입력
- 이미지 파일 업로드 input (`image/*`)
- 물품명 text input
- 보유 수량 number input
- 수량 선택 기능 checkbox
- 비밀번호 password input 2종

## 4) 사용자 흐름

### 일반 사용자
- 홈 → 대여 선택 → 물품 선택 → 수량 확인 → 사용자 정보 입력 → 완료
- 홈 → 반납 선택 → 대여중 항목 선택 → 반납 완료

### 관리자
- 홈 → 관리자 진입(인증 단계가 별도 존재할 가능성 있음) →
  - 물품 등록/삭제
  - 전체 기록 조회
  - 비밀번호 변경

### 상태 처리 흐름
- 처리 버튼 클릭 시 `처리 중...` 상태 노출
- 성공 시 전체 오버레이(체크 아이콘/안내문) 노출 후 메인으로 복귀

## 5) 모바일과 데스크톱 차이

관찰 가능한 클래스/패턴 기준:
- 모바일 우선(`p-4`, `text-sm`) + 데스크톱 확장(`md:p-6`, `md:text-base`)
- 카드 그리드:
  - 모바일 2열
  - 데스크톱 3열 또는 2열(관리 페이지)
- 아이콘/텍스트 크기 `md:` 접두로 단계적 확대
- 하단 고정 CTA는 모바일/데스크톱 모두 유지하되, 최대 폭 컨테이너(`max-w-3xl`) 중앙 정렬

## 6) 눈에 보이는 색감, 간격, 타이포 느낌

### 색감
- 기본 배경: 매우 연한 회색/화이트
- 핵심 액션(대여): **그린 계열** (`green-500`, `green-50`)
- 반납 액션: **레드 계열** (`red-500`, `red-50`)
- 관리자 보조 액션: 블루/퍼플/그레이 포인트

### 간격/레이아웃
- 카드형 UI, 둥근 모서리(라운드 큼: `rounded-xl~3xl`)
- 섹션 간 여백 충분 (`space-y-*`)
- 하단 고정 바 + 배경 블러(`backdrop-blur`) 사용

### 타이포
- 전반적으로 굵은 제목 + 중간/작은 본문 조합
- 버튼 텍스트는 굵고 크며 가독성 우선
- 한국어 라벨 중심, 일부 영어 보조 표기(예: Grade/Class)

---

## 7) 애니메이션/상호작용

- 카드 hover/active 시 그림자/배경 변화
- 버튼 active 시 살짝 축소(`active:scale-95`)로 눌림 피드백
- 전환 시 `transition-all`/`transition-colors` 다수 사용
- 성공 화면은 페이드 인 계열 애니메이션(`animate-in fade-in duration-300`)
- 비활성 버튼은 색상 다운 + 커서 제한으로 명확한 상태 구분

---

## 8) 확실한 내용 / 추정한 내용 / 아직 불확실한 내용

### 확실한 내용 (관찰 근거 강함)
- 대여/반납/관리의 3축 기능 구조가 존재함
- 대여는 최소 3단계(선택→수량→사용자) 멀티스텝 형태
- 반납은 현재 대여 항목 선택 후 일괄 반납
- 관리자 페이지에서 물품 CRUD 일부(추가/삭제) 및 기록 조회/비밀번호 변경 지원
- 하단 고정 CTA 패턴과 반응형 `md` 분기 사용

### 추정한 내용 (근거는 있으나 확정 어려움)
- 관리자 진입 전에 비밀번호 인증 화면이 별도로 있을 수 있음
- 홈 화면에서 "대여/반납/관리" 진입 버튼 3개가 메인 동선일 가능성 높음
- 데이터 저장소는 원격 DB(문서형)일 가능성이 높음

### 아직 불확실한 내용 (추가 관찰 필요)
- 최초 랜딩 화면의 정확한 헤더/브랜딩 문구
- 관리자 인증 UX(모달/전용 화면/세션 유지 방식)
- 오류 처리 문구 전체(네트워크 실패, 유효성 실패 케이스)
- 접근성 수준(키보드 탐색, 스크린리더 라벨링)
- 실제 배포 환경에서의 정확한 폰트 패밀리

## 부록: 구현 시 주의 포인트(분석 문서 관점)

- 본 문서는 "공개 UI 동작 재현"을 위한 레퍼런스이며, 원본 내부 로직/코드 구조 복제를 지시하지 않는다.
- 불확실 항목은 구현 전 별도 검증(실사용 클릭 테스트, 모바일 실기기 확인) 권장.
